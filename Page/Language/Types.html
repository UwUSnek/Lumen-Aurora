Data types define how values are interpreted during runtime and how they can be used within expressions.

Both languages are statically types.
Every value has an associated type, regardless of it being a right or a left value.
The type of each value must be known at compile time, as there are no runtime type informations.

There are no classes, strings, characters, matrices or vectors. The latter 2 are aliases of array types. See Standard Modules.

Lux and Umbra feature 7 primitive types, which can be used to define more complex types such as structs, arrays and tuples.

These types are always defined and they act as if they were declared in the core module.
Primitive types are not keywords and can be shadowed like any other symbol.

The primitive types are:

All primitives types are convertible to each other. This is explained in detail in Implicit conversions.

int, uint, long, ulong.
Integral types are used to represent integers.
Unsigned integers cannot represent negative numbers, but the maximum value is twice as high as their signed counterpart.

All signed integral types use the Two's Complement to represent negative numbers.
Positive numbers and 0 use the corresponding binary value in both signed and unsigned integral types.

float, double.
Floating point types are used to approximate real numbers.

Both floating point types are encoded according to the IEEE 754 Standard.
Positive and negative inf, nan and signed 0 are all supported.
+0 and -0 are equal.

bool.
The Boolean type is used to represend conditions.
Its value can be true or false.

Booleans purposely waste 31 bits for performance reasons.
Smaller types require additional bitwise operations and expensive conversions between the CPU and GPU data.

Enums are used to group named literals of the same type.
The base type of the enum can be specified after its name and it must be one of the primitive types.
Elements are declared in the same way as variables, but neither the type nor the const qualifier are specified explicitly.
The initializer values are implicitly converted to the base type of the enum.

Elements of integral enums which are not initialized have the value of the preceeding element + 1, or 0 if they are the first.
Elements of boolean and floating point enums must always be initialized.
Multiple elements can have the same value, but each element can only be declared once.
Empty enums, unnamed enums and trailing element ; are allowed.

Example 1
enum e : int { a, b; // Ok, a = 0, b = 1 c = -266250; // Ok, c = -266250 d, e = 0; // Ok, d = -266249, e = 0 }
Example 2
enum c : double { pi = 3.14159; // Ok e = 2.71828; // Ok //phi; // Error, uninitialized non-integral enum value } void main(){ c a = c.pi; // Ok c b = 3.14159; // Ok int c = a; // Ok, implicit conversion from c (double) to int }

By default, all enums behave exactly as their base types, much like a symbol alias.
When implicit conversions from other types are undesirable, enums can be declared as strict.
Strict enums only allow values of the same enum, but can still be implicitly converted to other types.

Example
strict enum c : double { pi = 3.14159; e = 2.71828; phi = 1.61803; } void main(){ c a = c.phi; // Ok //c b = 1.61803; // Error, cannot convert from double to strict enum int c = a; // Ok, implicit conversion from c (double) to int }

Structs are used to group named values of different types.
The members are declared in the same way as variables, and they can be of any type, including tuples, arrays and other structs.
They cannot be const and cannot have initializer values.
As with variables, uninitialized members cannot be used within expressions. Struct values must also be fully initialized before they can be used in expressions.

Empty structs, unnamed structs and trailing element ; are allowed.
Struct initializer values are explained in Constructors.

Arrays are used to group unnamedvalues of the same type.
Unlike other aggregate types, they don't need to be declared.
Arrays can use any base type, including structs, tuples and other arrays.
An array of arrays is called a multidimensional array. There is no limit to the number of dimensions an array can have.

Array symbols are declared by referencing the base type and specifying the length in [] tokens.
In variables and array parameters the [] can be left empty to let the inizializer value determine the length. When this syntax is used, the initializer value becomes required and must be an implicitly convertible array of the same length and number of dimensions.

Array types are determined by their lenght, base type and number of dimensions.
The length of each dimension must be known at compile time and cannot be 0.
In Lux, dynamic arrays are not supported. In Umbra, they are provided by the array standard module.
Element access is explained in Element indexing.

Tuples are used to group unnamed values of different types.
As with arrays, they don't need to be declared in order to be referenced.
Tuples can contain elements of any type, including arrays and other tuples.

Any sequence of types enclosed in () tokens is interpreted as a tuple type.
The type of a tuple is determined by the type and order of its elements. If these are identical to another tuple, the tuples have the same type.

Tuples cannot have 0 elements.
Element access is explained in Element indexing.

The void type is a special type. It is exclusively used to define functions that don't return any value.
It cannot store values like other types, but it can be used as value of typename symbols.

Passing viod to a template parameter is allowed.

Typenames are special types that can store the complete name of any type to allow type comparisons.
When a type is passed to a function parameter or anywhere a typename value is required, it is interpreted as a typename.
typename constructors are allowed, as well as typename struct elements, typename tuple elements and typename arrays.

All typename values must be known at compile time, and they cannot be used to declare symbols or call constructors (for these, templates are used).
Passing typename to a template parameter is allowed.
typename is a valid typename value.

Example
typename t1 = typename; // Ok t1 t2 = int; // Error, typename cannot be used to declare symbols auto main(){ return typename(typeof(main)) == t1; // Ok, returns false }

The auto specifier can be used as a type when declaring a symbol.
The actual type of the symbol is determined by the initializer value, if its is a variable, or by the return value if it is a function.

auto is not allowed in struct members declarations or function parameters.
auto constructors are equivalent to unnamed constructors.

Referencing syntax
auto
Example 1
auto func(){ return true; // func has type bool } auto main(){ auto a = 2; // a has type ulong auto b = 5.4 * a; // b has type double auto c = func(); // c has type bool // No return statement, main has type void }
Example 2
auto a = int[4](1, 1, 0, 1); // Ok, a has type int[4] and value (1, 1, 0, 1) auto const b = !a; // Ok, b has type bool[4] and value (false, false, true, false)

The typeof and baseof specifiers are used in the same way as auto, but the type is defined by the expression that is passed to them.
typeof returns the type of the expression, whereas baseof returns its base type.
Passing a type to typeof returns typename.

baseof can only be used on arrays, enums and primitive types.
The base type of primitive types is the type itself.
The values passed to these specifiers can be runtime.

The typeof and baseof specifiers can be used as constructors.

When an enum or struct doesn't have to be explicitly referenced more than once, unnamed types can be used to define both the type and the symbol in the same construct.
Similarly to the C syntax, the type identifier is not specified and the definition is used as type of the symbol.

Any symbol defined inside these types can still be referenced through the typeof and auto specifiers.
Elements of enums and structs can be indexed and accessed normally.
Defining an unnamed type without also declaring the symbol is not allowed.

Example 1
strict enum : bool { on = true; off = false; } grasses; void main(){ grasses = typeof(grasses).on; }
Example 2
struct { uint a, b; enum : int { idk1; idk2; } c; } fun(){ return (1, 1, 0); // Ok, unnamed constructor of unnamed struct } void main(){ auto var = fun(); // Both are allowed var.c = typeof(var.c).idk1; var.c = typeof(typeof(fun).c).idk2; typeof(var.a) a = 0; }
