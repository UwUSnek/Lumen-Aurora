Comments are portions of code that are intentionally ignored.
They can be used to provide informations or temporarily exclude lines of code.
Comments are allowed to contain any sequence of Unicode characters of any length.

There are two types of comments: single line and multiline.
Single line comments start with // and end at the next character (\n).
Multiline comments start with /* and end with */.
Not closing a comment is allowed. This will cause it to include everything from its starting character sequence to the end of the file.

Comments cannot be nested.
Because they are not code, they also cannot be referenced in any way.
Thought Preprocessor macros can process and copy comments, this has no effect whatsoever and is only useful if the preprocessed code needs to be read by external tools.

Syntax Example 1
// Single line comment /* Multi line comment */ /* Another multiline comment
Example 2
/* Multi line /* Another comment */ comment */ ^ Error, unexpected token "comment"
Example 3
/* Multi line /* Another comment */ comment */ ^ Error, unexpected token "comment" ^ Error, unexpected token "comment"
